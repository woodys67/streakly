# Streakly 스트릭 시스템 문서

## 1. 스트릭(Streak)이란?

스트릭은 챌린지를 **연속으로 완료한 일수**를 의미합니다. Streakly의 핵심 동기부여 메커니즘으로, 오늘부터 거슬러 올라가며 하루도 빠짐없이 체크인한 날수를 계산합니다.

---

## 2. 스트릭 계산 방식

### 2.1 `currentDay` 산출

```
currentDay = (오늘 날짜 - 챌린지 시작일).일수 + 1
             (최솟값: 1, 최댓값: targetDays)
```

챌린지 시작일로부터 경과한 날수에 1을 더한 값이 현재 진행 일차입니다.

### 2.2 스트릭 계산 (`challenge.dart:40-52`)

```dart
int get streak {
  if (completedDays.isEmpty) return 0;
  final today = currentDay;
  int count = 0;
  for (int d = today; d >= 1; d--) {
    if (completedDays.contains(d)) {
      count++;
    } else {
      break;  // 단 하루라도 빠지면 즉시 중단
    }
  }
  return count;
}
```

- `completedDays`: 완료된 '일차 번호' 목록 (예: `[1, 2, 3, 5, 6]`)
- 오늘 일차부터 역순으로 탐색하며 연속으로 완료된 일수를 셉니다
- **하루만 빠져도 0부터 다시 시작**

### 2.3 완료 조건

| 시나리오 | 완료 처리 시점 |
|----------|--------------|
| 서브루틴 없는 챌린지 | `toggleTodayComplete()` 호출 시 즉시 |
| 서브루틴 있는 챌린지 | **모든** 서브루틴 체크 시 자동으로 해당 일차 완료 처리 |

서브루틴이 있는 경우, 일부만 완료하면 그날은 완료로 인정되지 않습니다.

## 3. 데이터 저장 구조

### 3.1 로컬 저장 (비로그인 사용자)

`SharedPreferences`에 JSON 형태로 저장됩니다.

```json
{
  "completedDays": [1, 2, 3, 4, 5],
  "completedSubRoutines": {
    "3": ["sub-id-a", "sub-id-b"],
    "4": ["sub-id-a", "sub-id-b"]
  }
}
```

`completedDays`는 **날짜(Date)가 아닌 일차 번호(Day Number)**입니다. 챌린지가 2025-01-01에 시작했다면 Day 1 = 2025-01-01, Day 2 = 2025-01-02입니다.

### 3.2 클라우드 저장 (Supabase, 로그인 사용자)

| 테이블 | 칼럼 | 설명 |
|--------|------|------|
| `challenges` | `completed_days` (integer[]) | 완료된 일차 번호 배열 |
| `sub_routine_completions` | `day_number`, `sub_routine_id` | 서브루틴별 완료 기록 |

## 4. UI 표현

### 4.1 StreakCard (홈 화면)

- 현재 챌린지의 스트릭 일수를 🔥 아이콘과 함께 표시
- **최근 7일을 도트(●)로 시각화**: 완료일은 주황색 채움, 미완료일은 회색, 오늘은 테두리 강조
- `StreakCard`는 오늘 진행해야 할 챌린지 중 **첫 번째** 챌린지의 스트릭을 보여줌

### 4.2 totalStreak (ChallengeProvider)

```dart
int get totalStreak {
  return _challenges.fold<int>(0, (sum, c) => sum + c.streak);
}
```

현재 활성 챌린지 전체 스트릭의 **합산값**입니다. 단일 최장 스트릭이 아닌 모든 챌린지의 현재 스트릭을 더한 것입니다.

## 5. 스트릭 관련 배지

배지 판정에는 `_maxStreakAcrossAll()` — 모든 챌린지 중 **가장 긴 단일 스트릭** — 이 사용됩니다.

| 배지 ID | 이름 | 조건 |
|---------|------|------|
| `streak_001` | 첫 불꽃 | 스트릭 1일 |
| `streak_002` | 불씨 | 스트릭 3일 |
| `streak_003` | 작은 불꽃 | 스트릭 7일 |
| `streak_004` | 모닥불 | 스트릭 14일 |
| `streak_005` | 화톳불 | 스트릭 21일 |
| `streak_006` | 용광로 | 스트릭 30일 |
| `streak_007` | 불의 지배자 | 스트릭 50일 |
| `streak_008` | 태양 | 스트릭 66일 (습관 완전 정착) |
| `streak_009` | 영원한 불꽃 | 스트릭 100일 |
| `streak_010` | 불사조 🦅 | 스트릭이 끊긴 후 재시작하여 이전 최고 기록 경신 |
| `streak_011` | 부활 ↩️ | 스트릭 0에서 7일 이상 회복 |
| `streak_012` | 올해의 불꽃 📅 | 한 해 누적 완료일 300일 이상 |

> **불사조 / 부활 배지 탐지 방식**  
> 체크인 직전 스트릭이 0이었고 `completedDays`가 비어있지 않으면 `streakBrokeRecently = true`로 판정합니다. 이후 스트릭이 7일 이상 되면 '부활', 이전 최고 기록을 넘으면 '불사조' 배지가 해제됩니다.

## 6. 스트릭을 놓쳤을 때 — 현재 상태 및 복구 방안

### 6.1 현재 구현 상태

현재 Streakly에는 **스트릭 복구(Streak Freeze/Repair) 기능이 없습니다**. 하루를 놓치면 스트릭은 즉시 0으로 초기화됩니다. 단, 앱은 이를 실패로 취급하지 않고:

- 배지 시스템에서 '부활(streak_011)', '불사조(streak_010)' 배지로 재기를 **적극 격려**합니다
- `completedDays` 기록은 유지되므로 완료율(successRate)에는 계속 반영됩니다

### 6.2 도입 가능한 복구 메커니즘 (제안)

#### 옵션 A: 스트릭 프리즈(Streak Freeze) — 권장
> "오늘 하루 쉬어도 스트릭은 유지됩니다"

- 사용자에게 월 1~2회 '프리즈 토큰'을 제공
- 스트릭 계산 시 프리즈된 날은 완료로 취급
- 장점: Duolingo 방식으로 사용자 유지율이 검증된 방법
- 단점: 습관 형성이 목적인 앱의 '정직성'을 희석할 수 있음

#### 옵션 B: 하루 유예(Grace Period)
> "자정이 지나도 다음날 오전 3시까지 전날 체크인 허용"

- 구현 단순: `toggleTodayComplete` 호출 시 오전 3시 이전이면 전날 일차에 기록
- 장점: 늦은 밤 루틴 사용자에게 자연스러운 경험
- 단점: 악용 방지가 필요 (무한 소급 방지)

#### 옵션 C: 스트릭 회고 수정(Retroactive Edit)
> "달력에서 특정 날짜를 직접 체크 가능"

- PRO 기능으로 제한하거나 월 1회 무료 제공
- 장점: 여행, 병가 등 불가피한 상황 대응
- 단점: 데이터 신뢰성 저하 우려, 구현 복잡도 높음

#### 옵션 D: 미션 재도전(Makeup Challenge)
> "어제 못 했던 것을 오늘 2배로 체크인하면 복구"

- 현재 `completedDays` 구조상 중복 체크인이 불가하므로 별도 `makeupDays` 필드 필요
- 장점: 사용자에게 능동적 행동을 요구하여 습관 가치 유지
- 단점: 구조 변경 필요, UX 설계 복잡

## 7. 개별 챌린지 스트릭 vs. 전체 스트릭 — 설계 의견

### 현재 방식: 개별 챌린지 스트릭 + 단순 합산

```
홈 화면: 첫 번째 챌린지의 스트릭만 표시
ChallengeProvider.totalStreak: 모든 챌린지 스트릭의 합
배지 판정: _maxStreakAcrossAll() = 가장 긴 단일 스트릭
```

### 7.1 개별 챌린지 스트릭 표시의 장점

- **행동과 피드백의 직접 연결**: "운동 챌린지 5일 연속"이라는 메시지는 사용자가 어떤 행동을 유지해야 하는지 명확히 알려줌
- **여러 챌린지 동시 진행 시 공정성**: A 챌린지를 열심히 해도 B 챌린지를 놓치면 전체 스트릭이 깨지는 불합리함 없음
- **세밀한 개선 포인트 파악**: "독서 챌린지는 스트릭 12일, 운동은 2일" — 어디에 집중해야 할지 사용자가 스스로 파악 가능

### 7.2 전체(통합) 스트릭 표시의 장점

- **단순한 메시지**: "오늘도 모든 챌린지 완료 — 15일 연속!" 은 성취감이 크고 공유하기 좋음
- **앱 전체 충성도 측정**: 사용자가 앱을 며칠째 이용하고 있는지 앱 레벨에서 추적 가능
- **소셜/경쟁 기능 연계**: 미래의 팀 챌린지(PRO)에서 "팀 전체 스트릭"으로 확장하기 용이

### 7.3 권장 방향

**현재 구조(개별 표시)를 유지하되, 전체 스트릭 개념을 보조 지표로 추가하는 하이브리드 방식을 권장합니다.**

```
[홈 화면]
┌─────────────────────────────────┐
│ 🔥 오늘도 불타오르는 중!          │
│ 전체 15일 연속 (챌린지 2개 모두) │  ← 전체 스트릭 (조건: 오늘 모든 챌린지 완료)
└─────────────────────────────────┘

[챌린지 탭]
┌─────────────────────────────────┐
│ 운동하기  🔥 12일                │  ← 개별 스트릭
│ 독서하기  🔥 3일                 │
└─────────────────────────────────┘
```

- **전체 스트릭**: "오늘 진행해야 할 모든 챌린지를 완료한 날"을 기준으로 계산 → 앱 충성도의 상징적 지표
- **개별 스트릭**: 챌린지 탭 및 상세 화면에서 각 챌린지 진행 상황 파악용

이 방식은 Duolingo(전체 스트릭 중시)와 Habitica(개별 습관 추적)의 장점을 결합하며, 현재 코드 구조에서 `ChallengeProvider`에 `globalStreak` 게터 하나를 추가하는 것만으로 구현 가능합니다.
