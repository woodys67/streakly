## 1. 개요 및 핵심 콘셉트

### 한 줄 정의
> 친구·동료와 **같은 목표로 함께 달리고, 서로가 서로의 코치**가 되는 팀 습관 챌린지

### 왜 필요한가?

| 개인 챌린지의 한계 | 팀 챌린지가 해결하는 것 |
|---|---|
| 혼자라서 포기하기 쉬움 | 팀원이 보고 있다는 사회적 책임감 |
| 슬럼프 때 동기 부여 없음 | 응원 반응 & 독려 알림으로 재참여 유도 |
| 성취감이 개인에서 끝남 | 팀 전체 달성 시 특별 배지로 집단 성취감 |
| 스트릭이 혼자만의 기록 | 팀 스트릭 공유로 연대감 형성 |

### 핵심 지표 (목표)
- 팀 챌린지 참여자의 **21일 완주율 > 싱글 챌린지 완주율 × 1.5배**
- PRO 전환율: 팀 초대 수신자 → PRO 구독 전환 **20% 이상**

## 2. 기능 설계

### 2.1 팀 구성 & 초대

#### 팀 생성
- 팀 이름 입력 (최대 20자)
- 팀 이모지/아이콘 선택 (사전 정의된 16개 세트)
- 챌린지 목표 입력 (기존 싱글 챌린지 생성과 동일)
- 목표 일수 선택: 14일 / 21일 / 28일 / 커스텀
- 팀 공개 설정: 초대 링크 전용 (기본) / 코드 검색 가능

#### 초대 방식
| 방식 | 설명 |
|---|---|
| 공유 링크 | 딥링크 URL 1회 생성, 만료 시간 72시간 |
| 초대 코드 | 6자리 영숫자 코드 (예: `FIRE21`) |
| 연락처 공유 | iOS/Android 연락처 연동 (v2 이후 검토) |

#### 팀 규모
- **최소**: 2명
- **최대**: 10명 (PRO 기준, 추후 확장 검토)
- 팀장(생성자) 1명 + 팀원 최대 9명

#### 팀장 권한
- 챌린지 시작 일자 확정
- 팀원 강제 퇴장
- 팀 해산
- 챌린지 연장 (팀원 과반수 동의 시)

---

### 2.2 팀 챌린지 진행

#### 챌린지 동기화
- 팀원 전원 **동일한 챌린지 목표** 공유
- 각자의 서브 루틴은 개인 설정 가능 (목표만 공유)
- 시작일은 팀장이 확정, **동시 D1 스타트**
- 중도 참가 불가 (공정성 유지) — 단, 팀장 재초대 허용

#### 진행 상태 실시간 반영
- 팀원 각자의 오늘 체크인 여부 실시간 표시 (✅ / 미완료)
- 팀 전체 오늘 완료율 퍼센트 표시
- 팀 스트릭: 팀원 **전원**이 연속으로 완료한 날 수 카운트

#### 데이터 구조 (개요)
```
teams
  └─ id, name, icon, created_by, max_members, status

team_members
  └─ team_id, user_id, joined_at, role (leader/member)

team_challenges
  └─ team_id, challenge_name, target_days, start_date, status

team_daily_completions
  └─ team_challenge_id, user_id, day_number, completed_at
```

### 2.3 경쟁 & 동기부여 시스템

#### 리더보드
- **연속 달성일(스트릭)** 기준 팀 내 순위
- 동점 시: 오늘 체크인 시간 빠른 순
- 주간 리셋 없음 (챌린지 기간 전체 누적)

#### 응원 반응 시스템
| 반응 타입 | 설명 |
|---|---|
| 🔥 불꽃 | "불타오르고 있어!" — 응원 기본 반응 |
| 💪 파이팅 | "힘내!" |
| 👏 박수 | "잘했어!" — 당일 완료한 사람에게 |
| ⚡ 번개 | "빨리빨리!" — 미완료 팀원에게 독려 |

- 하루 팀원 1인당 **최대 3회** 반응 가능
- 반응 시 수신자에게 앱 내 알림 전송

#### 배지 시스템
| 배지 | 조건 |
|---|---|
| 🏆 팀 완주 | 팀원 전원 챌린지 완료 |
| 🔥 팀 스트릭 7일 | 팀 전원 7일 연속 달성 |
| 🔥🔥 팀 스트릭 14일 | 팀 전원 14일 연속 달성 |
| ⭐ 팀 MVP | 챌린지 종료 시 최고 완료율 1위 |
| 🌟 무결점 | 개인 기준 하루도 빠짐없이 완료 |

#### 팀 일지 (공유 로그)
- 기존 데일리 로그에 "팀에 공유" 옵션 추가
- 공유된 로그는 팀 챌린지 상세 화면의 팀 피드에 노출
- 비공개 로그는 개인에게만 표시 (기존 동일)

### 2.4 알림 시스템

| 알림 종류 | 발송 시점 | 수신 대상 |
|---|---|---|
| 팀원 가입 | 새 팀원 참가 시 | 기존 팀원 전체 |
| 팀 챌린지 시작 | 시작일 D-1 저녁 8시 | 팀원 전체 |
| 팀원 완료 알림 | 팀원 체크인 시 | 나를 제외한 팀원 |
| 독려 알림 | 오후 8시 기준 미완료자 존재 시 | 미완료 팀원 |
| 응원 반응 수신 | 반응 발생 시 즉시 | 반응 수신자 |
| 팀 스트릭 달성 | 전원 완료 확정 시 | 팀원 전체 |
| 챌린지 완료 | D+1 | 팀원 전체 |

## 3. UX 플로우

### 팀 생성 플로우
```
[챌린지 탭]
    └─ [+ 새 챌린지] 버튼
        └─ 모드 선택: [싱글] / [팀 (PRO)]
            └─ (미구독 시) PRO 업그레이드 바텀시트
            └─ (PRO 유저) 팀 정보 입력
                └─ 팀 이름 / 아이콘 설정
                    └─ 챌린지 목표 & 기간 설정
                        └─ 초대 링크/코드 공유
                            └─ 팀원 대기 화면
                                └─ [챌린지 시작] (팀장만)
```

### 팀 참가 플로우 (초대 수신자)
```
[초대 링크 클릭 or 코드 입력]
    └─ 앱 미설치 → 스토어 이동
    └─ 앱 설치/실행 → 팀 정보 미리보기 화면
        └─ [참가하기] 버튼
            └─ 비로그인 → 로그인/회원가입 유도
            └─ 로그인 완료 → 팀 합류 확정
                └─ 팀 대기 화면으로 이동
```

## 6. 기술 설계

### 아키텍처

```
Flutter App
    ├── TeamChallengeProvider (Provider)
    │       ├── loadTeamChallenges()
    │       ├── createTeam()
    │       ├── joinTeam(code)
    │       ├── checkIn(teamChallengeId)
    │       └── sendReaction(toUserId, type)
    │
    └── Supabase
            ├── teams
            ├── team_members
            ├── team_challenges
            ├── team_daily_completions
            └── team_reactions
```

### 실시간 동기화

- **Supabase Realtime** (PostgreSQL CDC) 사용
- 구독 채널: `team_daily_completions` 테이블의 팀별 변경사항
- 팀원 완료 시 → 실시간 UI 업데이트 (새로고침 불필요)

```dart
supabase
  .from('team_daily_completions')
  .stream(primaryKey: ['id'])
  .eq('team_challenge_id', teamChallengeId)
  .listen((data) => updateMemberStatus(data));
```

### 딥링크 (초대 링크)

- Android: `streakly://invite?code=FIRE21`
- iOS: Universal Link + `streakly://invite?code=FIRE21`
- 미설치 시: App Store / Play Store로 리다이렉트 (Branch.io 또는 Firebase Dynamic Links 검토)

### 푸시 알림

- **FCM (Firebase Cloud Messaging)** + Supabase Edge Functions
- 팀 이벤트 발생 시 Edge Function에서 FCM 토큰으로 발송
- 알림 카테고리별 토글 설정 (기존 설정 화면에 추가)

### PRO 구독 게이트

```dart
// TeamChallengeProvider
Future<bool> canCreateTeam() async {
  final isPro = await subscriptionService.isProUser();
  return isPro;
}

// UI
if (!await provider.canCreateTeam()) {
  showProUpgradeBottomSheet(context);
  return;
}
```
